=== accessToken 사용 방법
일반적인 API 요청 시, `access_token` 헤더를 통해 인증된 사용자임을 증명할 수 있습니다. 모든 보호된 리소스에 대한 요청은 유효한 엑세스 토큰이 필요합니다.

=== Request
아래의 예제는 `access_token` 헤더를 포함하여 `GET` 요청을 보내는 방법을 보여줍니다.

임의의 요청이며 로그인 이후 모든 api에  `access_token` 헤더에 엑세스 토큰을 담아 요청해주세요.

[source,http]
----
GET /v1/weathers/all HTTP/1.1
Host: localhost:8080
access_token: eyJhbGciOiJIUzI1NiJ9.eyJzdWIiOiIxMjM0NTY3ODkwIiwiaWF0IjoxNTE2MjM5MDIyfQ.SflKxwRJSMeKKF2QT4fwpMeJf36POk6yJV_adQssw5c
----

=== Response (Success)
인증이 성공적으로 이루어지면, 서버는 상태 코드 200과 함께 요청된 데이터를 응답으로 반환합니다.

[source,json]
----
"HTTP/1.1" 200 "OK"
"Content-Type": application/json

{
  "data": [
    "// ... 여기에 성공적인 응답 데이터가 포함됩니다 ..."
  ]
}
----

=== Response (Unauthorized)
엑세스 토큰이 유효하지 않거나 만료된 경우, 서버는 상태 코드 4xx 번대 응답을 반환합니다.

[source,json]
----
"HTTP/1.1" 4xx "유효하지 않은 토큰"
"Content-Type": application/json

{
  "메시지": "JWT signature does not match locally computed signature. JWT validity cannot be asserted and should not be trusted.",
    "예외": "잘못된 토큰"
}
----
